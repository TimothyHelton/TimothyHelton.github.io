Python Project Generator

~Bill Gates


Background

I found myself writing various pieces of code, primarily in Python, for the K2 Data Science Bootcamp (which I highly recommend), and wanted to make them easy to share with the community.

As long as everything is in the right place creating Python packages and hosting them on PIP is a piece of cake.

The bash script pypackage_generator.sh will automate the setup process for creating a Python package, which uses SPHINX for documentation and Git for version control.

Thanks for looking around and enjoy the day!

Installation

All you need is the pypackage_generator.sh .

  1. go to source file
  2. Save as...
  3. Change permissions to make the file user executable 
                    chmod u+x pypackage_generator.sh

Quick Start Guide

Configuration

  1. Edit the pypackage_generator.sh file
  2. Enter your name in the author argument 
                                    AUTHOR="Timothy Helton"
  3. Enter your email address in the email argument 
                                    EMAIL="timothy.j.helton@gmail.com"
  4. Enter the current year 
                                    YEAR="2017"

Execution

The script has a single required argument, which is the name of the package. 

                    ./pypackage_generator.sh example_package

What will the script generate?

  • Package Main Directory
  • License File
  • The default license is based off the open source license used by NumPy.
  • Manifest File
  • Readme File
  • Python Setup File
  • .gitignore File
  • Data Directory
  • Documentation Directory
  • Source Directory
  • Source Constructor
  • Tests Directory
  • Tests Constructor
  • .git Directory
                    ├── example_package
                         ├── LICENSE.txt
                         ├── MANIFEST.in
                         ├── README.md
                         ├── data/
                         ├── docs/
                         │   ├── _build
                         │   ├── _static
                         │   └── _templates
                        ├── example_package/
                        │   ├── __init__.py
                        │   └── tests/
                        │       └── __init__.py
                        ├── notebooks
                        ├── setup.py
                        ├── .git/
                        └── .gitignore

NextSteps

  1. Activate your preferred Python interpreter
  2. Create a new virtual environment
  3. Navigate to the new package main directory
  4. Install your package in developer mode 
                                    pip install -e .
  5. Go to the docs directory and configure SPHINX 
                                    sphinx-quickstart
